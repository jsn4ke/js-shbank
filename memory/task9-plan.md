# Task 9: 主流程集成计划

## 目标
实现 main.py 主入口，集成所有模块完成完整的爬取和存储流程。

## 小功能点清单

### 9.1 创建 main.py 入口文件
- 定义主函数入口
- 处理命令行参数（如使用配置文件路径）
- 添加日志配置

### 9.2 实现数据获取流程
- 初始化 ApiClient
- 调用 fetch_all_products 获取所有产品
- 处理网络错误和重试

### 9.3 实现数据解析流程
- 初始化 Parser
- 解析 API 响应数据
- 转换为 ProductModel 列表

### 9.4 实现数据存储流程
- 初始化 CsvRepository
- 批量保存产品数据
- 处理存储错误

### 9.5 实现年化计算流程（可选）
- 初始化 YieldCalculator
- 计算各期限年化收益率
- 更新 ProductModel 数据

### 9.6 添加错误处理
- 捕获并处理所有可能的异常
- 提供用户友好的错误提示
- 记录详细的错误日志

### 9.7 添加进度显示
- 显示当前爬取进度（分页数）
- 显示成功/失败统计
- 显示保存的记录数

### 9.8 编写测试
- 测试主流程正常执行
- 测试网络错误处理
- 测试存储错误处理
- 测试空数据情况

## 开发顺序
1. 写测试 → 测试失败 (RED)
2. 写代码 → 测试通过 (GREEN)
3. 重构 → 代码优化 (IMPROVE)

## 验收标准
- [ ] 所有测试通过
- [ ] 测试覆盖率 ≥ 80%
- [ ] 主流程能够正常执行
- [ ] 错误处理完善
- [ ] 日志记录完整
- [ ] 进度显示清晰

## 架构设计

```
main.py
├── 加载配置
├── 初始化组件
│   ├── ApiClient
│   ├── Parser
│   ├── CsvRepository
│   └── YieldCalculator (可选)
├── 获取数据
│   ├── fetch_all_products
│   └── 错误处理与重试
├── 解析数据
│   ├── parse_response
│   └── 字段映射
├── 存储数据
│   ├── save_batch
│   └── 错误处理
└── 输出统计
    ├── 总产品数
    ├── 成功数
    └── 失败数
```

## 命令行接口

```bash
# 默认运行（使用配置文件）
python src/main.py

# 指定输出目录
python src/main.py --output-dir ./data

# 指定配置文件
python src/main.py --config ./config.toml
```
